# 📱 GAFAM Android Relay Agent (A-Relay)

The **GAFAM Android Relay Agent** is a minimalist, battery-efficient Android application designed to run persistently in the background. Its primary purpose is to bypass traditional closed operators by turning your actual mobile phone into an automated, encrypted SMS and Contact gateway that synchronizes in real-time with your private GAFAM VPC server (Ubuntu Docker/Cloudron).

**CRITICAL RULE:** The Android app communicates **ONLY** with your personal VPC. It never talks to the global frontend domain. This connection can be further secured using an integrated Outline VPN relay on the VPC side.

## 🔒 The "Big White Button" Pairing Protocol

To establish a secure, definitive cryptographic handshake between the Android App and the web platform without requiring standard passwords, we use a simple physical-pairing gesture:

1. **VPC IP Discovery:** The user inputs their private GAFAM VPC local URL/IP (e.g. `http://192.168.1.50:5150`) and their telephone number into the Android App.
2. **P2P Challenge Broadcast:** The web dashboard displays a temporary, pulsed pairing challenge token (`P2P_AUTH_74B9D0E2`) generated by the Loco Rust backend.
3. **The Big White Button Tap:**
   * The Android App interface displays a single **Big White Circular Button** on a sleek dark-themed background.
   * When tapped, the app securely broadcasts a verification payload containing the phone's unique hardware signature to the VPC server.
   * The server validates the match, marks the phone status as `AUTHORIZED`, and locks the cryptographic bridge. All subsequent incoming SMS and contacts will now sync automatically.

## ⚙️ Core Components & Android Services

To build the relay agent tomorrow, the app will require four main files and configurations:

### 1. Android Manifest Permissions (`AndroidManifest.xml`)
The agent requires explicit system-level access to intercept text messages and query the device directory:
```xml
<!-- Manifest declarations -->
<uses-permission android:name="android.permission.RECEIVE_SMS"/>
<uses-permission android:name="android.permission.READ_SMS"/>
<uses-permission android:name="android.permission.READ_CONTACTS"/>
<uses-permission android:name="android.permission.INTERNET"/>
<uses-permission android:name="android.permission.FOREGROUND_SERVICE"/>
```

### 2. SMS Interceptor (`SMSReceiver.kt`)
A `BroadcastReceiver` that triggers on `android.provider.Telephony.SMS_RECEIVED` events. It parses incoming SMS metadata and forwards it instantly even if the device is locked:
```kotlin
class SMSReceiver : BroadcastReceiver() {
    override fun onReceive(context: Context, intent: Intent) {
        if (intent.action == Telephony.Sms.Intents.SMS_RECEIVED_ACTION) {
            val messages = Telephony.Sms.Intents.getMessagesFromIntent(intent)
            for (sms in messages) {
                val sender = sms.originatingAddress ?: "Unknown"
                val body = sms.messageBody ?: ""
                
                // Immediately dispatch via the HTTP WorkManager
                dispatchSmsToVpc(context, sender, body)
            }
        }
    }
}
```

### 3. Real-time Contacts Watcher (`ContactsObserver.kt`)
A `ContentObserver` bound to `ContactsContract.Contacts.CONTENT_URI`. Whenever the user creates, updates, or deletes a contact on their mobile phone, the watcher immediately sends the changes to the GAFAM VPC SQLite database.

### 4. Resilient Worker (`SyncWorker.kt`)
Uses Android's native `WorkManager` with `NetworkType.CONNECTED` constraints. If the device briefly loses Wi-Fi or cellular connectivity, all intercepted SMS messages are safely cached in the phone's local Room database and re-queued for transmission as soon as connectivity is restored.
